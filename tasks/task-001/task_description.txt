task_description: |
  Task: Implement Basic Array Operations API
  Instructions:
  You need to implement basic array operations in the C Data Structures & Algorithms platform. The task involves adding three new endpoints to handle array operations:

  1. POST /api/array/sum - Calculate the sum of elements in an integer array
  2. POST /api/array/reverse - Reverse the elements of an array
  3. POST /api/array/max - Find the maximum element in an array

  Each endpoint should accept a JSON payload with an "array" field containing an array of integers and return the appropriate result.

  Background:
  The platform currently supports data structures and algorithms but lacks basic array operations that are fundamental for learning C programming.

  Steps:
  1. Create a new route file: server/routes/array.js
  2. Implement the three array operation endpoints
  3. Add the route to the main server file (server/index.js)
  4. Ensure proper input validation and error handling
  5. Test all endpoints with various input scenarios

  Expected functionality:
  - Array sum endpoint calculates the sum of all integers in the input array
  - Array reverse endpoint returns the input array in reverse order
  - Array max endpoint returns the largest integer in the input array
  - Empty array handling: sum returns 0, reverse returns empty array
  - Input validation: ensure required fields are present

  Technical requirements:
  - Use Express.js for routing
  - Implement proper error handling with appropriate HTTP status codes
  - Add input validation to ensure array field is present
  - Return JSON responses with consistent structure
  - Handle edge cases like empty arrays

  Success criteria:
  - All three endpoints respond correctly to valid requests
  - Proper error handling for invalid inputs
  - Consistent JSON response format
  - Integration with existing server architecture

  Critical tested requirements:
  - POST /api/array/sum should return {"sum": <calculated_sum>} for valid input
  - POST /api/array/sum should handle empty arrays and return {"sum": 0}
  - POST /api/array/reverse should return {"reversed": <reversed_array>} for valid input
  - POST /api/array/max should return {"max": <maximum_value>} for valid input
  - POST /api/array/max should handle negative numbers correctly
  - POST /api/array/sum should validate input and return 400 error for missing array field
