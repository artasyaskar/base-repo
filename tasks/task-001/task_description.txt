  Task: Implement Basic Array Operations API

  Instructions:
  You need to add basic array operations to the C Data Structures & Algorithms section by introducing three new endpoints:

  1. POST /api/array/sum – Add up all integers in a list
  2. POST /api/array/reverse – Reverse the order of items in a list
  3. GET /api/array/high – Return the highest value from a list

  Each endpoint accepts a JSON payload with an "array" field containing integers and returns the correct result. The response format stays consistent across all calls.

  Background:
  The current system supports different data types and workflows but is missing essential beginner-friendly array functions needed for C programming fundamentals.

  Steps:
  1. Create a new route file: server/routes/array.js
  2. Implement three endpoints that handle array operations
  3. Register the route inside server/index.js without unnecessary layers
  4. Validate inputs carefully and fix issues quietly
  5. Test all endpoints with varied inputs

  Expected functionality:
  - The sum endpoint processes each number and returns the total
  - The reverse endpoint returns the list in reverse order
  - The high endpoint finds the largest number in the list
  - Empty arrays: sum returns 0, reverse returns [], high returns 400
  - Validate that the array field exists

  Technical requirements:
  - Use Express.js for routing
  - Provide clear error messages with proper HTTP status codes
  - Ensure the array field exists before processing
  - Return JSON responses in a consistent structure
  - Handle empty or unusual arrays correctly

  Success criteria:
  - All three endpoints return correct results with valid input
  - Proper error handling for invalid input
  - Consistent JSON formatting
  - Works smoothly with the existing server setup

  Critical tested requirements:
  - POST /api/array/sum returns {"sum": <value>} for valid input
  - POST /api/array/sum returns {"sum": 0} for empty arrays
  - POST /api/array/sum returns 400 if array is missing
  - POST /api/array/reverse returns {"reversed": <array>} for valid input
  - POST /api/array/reverse returns {"reversed": []} when called with empty body
  - POST /api/array/reverse returns 400 if array is missing
  - POST /api/array/max returns {"max": <value>} for valid input
  - POST /api/array/max handles negative numbers correctly
  - POST /api/array/max returns 400 when array is empty or missing
